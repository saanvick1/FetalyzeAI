#### Preprocessing and encoding
- **Numerical features**: standardize using training set mean/std; persist scalers. For gestational age, consider both raw weeks and a quadratic term (weeks and weeks\(^2\)) to capture nonlinearity.
- **Categorical features**: use learned embeddings for high-cardinality (e.g., parity, oxytocin dose category) and one‑hot for low-cardinality booleans.
- **Comorbidity bitmask**: keep as binary vector; feed into MLP.
- **Missingness indicators**: for every clinical field, add a binary flag `is_missing_X`. Missingness itself is predictive in clinical data.
- **Signal quality**: compute a continuous score (0–1) and also bucket into tertiles (poor/medium/good) for subgroup analysis. Persist thresholds used during training.

#### Model architecture integration
- **Clinical encoder**: small MLP with residual connections:
```python
class ClinicalEncoder(nn.Module):
    def __init__(self, num_numeric, cat_embed_dims):
        super().__init__()
        self.embeds = nn.ModuleList([nn.Embedding(n, d) for n,d in cat_embed_dims])
        self.fc1 = nn.Linear(num_numeric + sum(d for _,d in cat_embed_dims) + num_binary, 128)
        self.bn1 = nn.BatchNorm1d(128)
        self.fc2 = nn.Linear(128, 64)
    def forward(self, numeric, cat_idxs, binary):
        x = [emb(cat_idxs[:,i]) for i,emb in enumerate(self.embeds)]
        x = torch.cat([numeric, torch.cat(x,dim=1), binary], dim=1)
        x = F.relu(self.bn1(self.fc1(x)))
        return F.relu(self.fc2(x))
```
- **Fusion**: concatenate `ctg_repr` (from your CTG encoder) and `clinical_repr` and pass to classifier head. Use layer normalization and dropout before final layers.

#### Training tips

### 2. Encode Guideline Knowledge as Deterministic and Soft Features
#### Deterministic FIGO/NICHD features
Implement a **feature engine** that computes FIGO-like features from raw CTG traces. For each record compute:
- **baseline_fhr**: median FHR over baseline window (algorithm: robust median after removing decelerations/accelerations using thresholding).  
- **variability_ms**: short-term variability estimate (e.g., SD of beat-to-beat or 1s window).  
- **variability_category**: {reduced, normal, increased} using clinically validated cutoffs.  
- **acceleration_count**: count of accelerations in last 10 min.  
- **late_decel_count**: count of late decelerations (onset after contraction peak) in last 30 min.  
- **variable_decel_count**: count of variable decelerations.  
- **prolonged_decel_flag**: boolean for decels > 2 min.  
- **contraction_rate_per_10min**: computed from toco or uterine activity channel.  
- **sinusoidal_pattern_flag**: boolean using spectral/shape matching.  
- **FIGO_category_estimate**: map rules to category 1/2/3.

Implement these as **deterministic functions** in the feature extraction pipeline (e.g., `extract_figo_features(raw_ctg)`), unit test them against textbook examples, and persist outputs.

#### Soft constraints and loss penalties
- **Rule‑aware regularizer**: add a penalty term to the loss when model predictions contradict strong guideline signals. Example:
  - If `FIGO_category_estimate == 3` and model predicts `normal` with probability > 0.6, add penalty \(\lambda \cdot \text{BCE}(\hat{p}_{normal}, 1)\). Tune \(\lambda\) small (0.1–1.0) to avoid overpowering data.
- **Auxiliary heads**: add auxiliary outputs that predict FIGO features from the fused embedding (multi-task). This forces the model to encode guideline concepts internally:
```python
loss = main_loss + alpha * sum(aux_losses)
```
Set `alpha` via validation.

### 3. Temporal and Workflow Awareness
#### Data representation
- **Sliding windows**: represent each record as a sequence of windows (e.g., 5‑minute windows with 1‑minute stride) covering the last 60 minutes. Each window has CTG features + snapshot clinical context (some clinical fields are static; others like oxytocin dose can be time-varying).
- **Timestamps**: keep absolute timestamps for each window to allow time‑aware positional encoding.

#### Model choices
- **Temporal encoder options**:
  - **Temporal CNN**: 1D convs across windows for efficiency.
  - **Transformer encoder**: use relative positional encodings; good for long-range dependencies.
  - **Bi‑directional LSTM**: simpler, lower compute.
- **Recommended**: Transformer encoder with small depth (2–4 layers) and attention heads (4–8) for interpretability of which windows drive decisions.

#### Implementation sketch
```python
# per-window: ctg_window_features -> ctg_window_repr (CNN/MLP)
# clinical per-window -> clinical_window_repr
sequence = torch.cat([ctg_window_repr, clinical_window_repr], dim=-1)  # shape (B, T, D)
sequence = sequence + time_positional_encoding(timestamps)
transformer_out = TransformerEncoder(sequence)  # (B, T, D)
pooled = attention_pool(transformer_out)  # weighted sum across time
logits = classifier_head(pooled)
```
- **Attention pooling**: compute attention weights across time windows and expose them in UI as “which minutes mattered”.

#### Decision framing
- **Predict both**:
  - **Immediate escalation risk** (binary within next 10–15 minutes).  
  - **Trajectory score** (slope of risk over next 30–60 minutes).  
Train with labels derived from objective outcomes (see section 5).

### 4. Uncertainty, OOD Detection, and Human‑in‑the‑Loop
#### Uncertainty estimation
- **Ensembles**: train 5–10 model seeds (same architecture, different initializations) and average predictions; use variance across ensemble as epistemic uncertainty.
- **MC Dropout**: enable dropout at inference and run \(N\) forward passes (e.g., \(N=30\)) to estimate predictive distribution.
- **Calibration**: apply temperature scaling on validation set to calibrate probabilities.

**Output**: return `p_compromise`, `epistemic_uncertainty` (ensemble variance), `aleatoric_uncertainty` (predictive entropy), and `calibrated_probability`.

#### OOD detection
- **Feature-space OOD detector**: train an IsolationForest or OneClassSVM on fused embeddings from training data. At inference, compute OOD score and threshold to flag records that are out-of-distribution.
- **Mahalanobis distance**: compute Mahalanobis distance in embedding space to class centroids; high distance → OOD.
- **Signal-quality OOD**: if `signal_quality_score < threshold`, mark as low‑confidence regardless of model output.

### 5. Training, Labeling, and Evaluation Strategy
#### Labels and outcomes
- **Prefer objective outcomes**: umbilical artery pH < 7.05, base deficit, 5‑minute Apgar < 7, NICU admission for respiratory or neurologic reasons. Create composite endpoints for “clinically meaningful compromise”.
- **Label windows**: map outcomes back to time windows (e.g., label a window positive if compromise occurs within 60 minutes after that window).

#### Losses and class imbalance
- **Loss function**: focal loss or weighted BCE to prioritize recall for compromise class. For clinical safety, tune for high sensitivity while controlling false alarms per hour.
- **Metric optimization**: use F\(_\beta\) with \(\beta=2\) (emphasize recall) for threshold selection, as you planned.   [Current page](citation-section://2039068526/4)
- **Resampling**: use stratified oversampling (SMOTE for tabular FIGO features) but avoid synthetic signal generation for raw traces unless validated.

#### Cross‑validation and external validation
- **Temporal split**: train on earlier time periods, validate on later periods to simulate deployment drift.
- **Site split**: if multi‑center data available, hold out entire hospitals for external validation.
- **Metrics to report**:
  - Sensitivity (recall) for compromise.
  - False alarms per hour (FA/hr).
  - Precision, AUROC, AUPRC.
  - Calibration plots and Brier score.
  - Coverage vs. accuracy for uncertainty thresholds (what fraction of cases are flagged as OOD/uncertain and how many of those contain true positives).

#### Adversarial and stress testing
