# 3. Smart training strategy that avoids bias and overfitting

## A. Use record-level splitting

Because each CTG record can produce many windows, never let windows from the same record appear in both train and test.

Use this:

```text
record_id 1001 → train only
record_id 1002 → validation only
record_id 1003 → test only
```

Never this:

```text
1001 window 1 → train
1001 window 2 → test
```

That would leak patient/record information.

---

## B. Use three splits, not two

Use:

```text
70% train
15% validation
15% test
```

Train set:

* model fitting
* feature scaling
* imputation
* augmentation

Validation set:

* early stopping
* threshold tuning
* temperature scaling
* choosing model weights

Test set:

* final metrics only

Do **not** tune on the test set.

---

## C. Do not create labels from your own features

This is very important.

Your current `train_ctu_signal_model.py` uses pH/Apgar first, but if those are missing, it falls back to fetal reserve score and deceleration burden to assign labels. That creates circular logic because the same features are then used to train the model.

Do not do this:

```python

    return 2
```

That means the model is partly learning labels that were created from its own input features.

Instead, labels should come only from clinical outcomes:

```python
def assign_risk_label(row):

    return -1  # unknown, exclude from supervised training
```

Then drop unknown labels:


This makes your model much more honest.

## Module 1: SignalPulse Encoder

Use a small CNN-TCN.

```python
class SignalPulseEncoder(nn.Module):
    def __init__(self, in_channels=3, hidden=48):
        super().__init__()
        self.net = nn.Sequential(
            nn.Conv1d(in_channels, 32, kernel_size=9, padding=4),
            nn.BatchNorm1d(32),
            nn.GELU(),
            nn.Dropout(0.15),

            nn.Conv1d(32, hidden, kernel_size=7, padding=3, dilation=1),
            nn.BatchNorm1d(hidden),
            nn.GELU(),

            nn.Conv1d(hidden, hidden, kernel_size=7, padding=6, dilation=2),
            nn.BatchNorm1d(hidden),
            nn.GELU(),

            nn.Conv1d(hidden, hidden, kernel_size=7, padding=12, dilation=4),
            nn.BatchNorm1d(hidden),
            nn.GELU(),

            nn.AdaptiveAvgPool1d(1)
        )

    def forward(self, x):
        # x shape: batch, channels, time
        return self.net(x).squeeze(-1)
```

## Module 3: Reliability Fusion Head

Use a small MLP that combines:

```text
signal embedding
+
engineered clinical features
+
XGBoost probabilities
+
signal quality score
```

Example:

```python
class ReserveFusionHead(nn.Module):
    def __init__(self, signal_dim, feature_dim, n_classes=3):
        super().__init__()

        self.gate = nn.Sequential(
            nn.Linear(signal_dim + feature_dim + n_classes, 64),
            nn.GELU(),
            nn.Linear(64, 3),
            nn.Sigmoid()
        )

        self.classifier = nn.Sequential(
            nn.Linear(signal_dim + feature_dim + n_classes + 3, 96),
            nn.BatchNorm1d(96),
            nn.GELU(),
            nn.Dropout(0.25),
            nn.Linear(96, 48),
            nn.GELU(),
            nn.Dropout(0.15),
            nn.Linear(48, n_classes)
        )

    def forward(self, signal_emb, features, xgb_probs):
        fused = torch.cat([signal_emb, features, xgb_probs], dim=1)
        gate = self.gate(fused)
        out = self.classifier(torch.cat([fused, gate], dim=1))
        return out, gate
```

This gives you a unique “gated clinical fusion” architecture without overcomplication.

## 4. Create a new file: `reservenet_model.py`

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class ReserveFusionMLP(nn.Module):
    def __init__(self, n_features, n_classes=3, hidden=96):
        super().__init__()

        self.feature_encoder = nn.Sequential(
            nn.Linear(n_features + n_classes, hidden),
            nn.BatchNorm1d(hidden),
            nn.GELU(),
            nn.Dropout(0.25),
            nn.Linear(hidden, 48),
            nn.GELU(),
            nn.Dropout(0.15),
        )

        self.reserve_head = nn.Linear(48, 1)
        self.class_head = nn.Linear(48, n_classes)

    def forward(self, features, xgb_probs):
        x = torch.cat([features, xgb_probs], dim=1)
        h = self.feature_encoder(x)
        logits = self.class_head(h)
        reserve_pred = torch.sigmoid(self.reserve_head(h)) * 100
        return logits, reserve_pred
```

This is your **unique architecture file**.

It is not overcomplicated. It outputs:

* risk class
* fetal reserve estimate

```python
import numpy as np
from sklearn.metrics import (
    accuracy_score, balanced_accuracy_score, f1_score,
    recall_score, precision_score, roc_auc_score,
    average_precision_score, confusion_matrix, brier_score_loss
)
from sklearn.preprocessing import label_binarize

def expected_calibration_error(y_true, probs, n_bins=10):
    preds = probs.argmax(axis=1)
    conf = probs.max(axis=1)
    correct = (preds == y_true).astype(float)

    ece = 0.0
    bins = np.linspace(0, 1, n_bins + 1)

    for i in range(n_bins):
        mask = (conf >= bins[i]) & (conf < bins[i + 1])
        if mask.sum() > 0:
            ece += mask.mean() * abs(correct[mask].mean() - conf[mask].mean())

    return float(ece)

def compute_all_metrics(y_true, probs):
    preds = probs.argmax(axis=1)
    y_bin = label_binarize(y_true, classes=[0, 1, 2])

    return {
        "accuracy": float(accuracy_score(y_true, preds)),
        "balanced_accuracy": float(balanced_accuracy_score(y_true, preds)),
        "macro_f1": float(f1_score(y_true, preds, average="macro", zero_division=0)),
        "weighted_f1": float(f1_score(y_true, preds, average="weighted", zero_division=0)),
        "high_risk_recall": float(recall_score(y_true, preds, labels=[2], average="macro", zero_division=0)),
        "watch_recall": float(recall_score(y_true, preds, labels=[1], average="macro", zero_division=0)),
        "ece": expected_calibration_error(y_true, probs),
        "confusion_matrix": confusion_matrix(y_true, preds).tolist(),
    }
```

# 13. The exact smart training pipeline

Use this flow:

```text
1. Load uploaded CTU ZIP
2. Verify real .hea/.dat files exist
3. Load FHR + UC signals
4. Extract record-level and window-level CTG features
5. Assign labels from pH/Apgar/base deficit only
6. Split by record_id
7. Fit imputer/scaler on training records only
8. Train regularized XGBoost
9. Train small ReserveFusionMLP using validation set
10. Calibrate probabilities on validation set
11. Evaluate once on untouched test set
12. Save model artifacts and result JSON
13. App loads saved results and displays stats
```

That gives you:

* stats in `app.py`
* no fetal_health dependency
* less overfitting
* no synthetic primary training
* actual CTU waveform use
* unique but defensible architecture
