# FetalyzeAI - Fetal Health Classification Dashboard

## Overview
A React + Vite dashboard for fetal health classification trained on **real intrapartum waveform data** from the CTU-UHB and CTGDL databases. The model is trained on raw FHR/UC signals — not tabular feature extracts. Run `python train_adaptive.py` to generate real evaluation metrics into `ctu_model_results.json`.

**Key Feature:** All training uses real hospital-grade waveform data (CTU-UHB, PhysioNet).

### Datasets (primary sources only)
- **CTU-UHB Intrapartum CTG Database (PRIMARY)**: 552 real intrapartum CTG recordings (FHR + UC @ 4 Hz). Clinical outcomes: cord blood pH, base deficit, Apgar scores. Source: PhysioNet (Chudáček et al., 2014)
- **CTU-CHB Annotation Dataset (CTGDL)**: Expert morphological annotations — baseline, decelerations, accelerations, uterine contractions. Source: Zenodo 19510407

### Model Architecture — AdaptiveReserveNet v3.0
- **4 domain experts** — Baseline FHR (LogReg), Variability+Spectral (GBM), Event Patterns (RF), **Temporal Trends** (LogReg on last-30-min delta features) ← new in v3
- **Attention-gated fusion** — learnable MLP that routes each sample to the most relevant expert (replaces heuristic confidence weighting from v2)
- **CNN-TCN PulseEncoder** — raw waveform embeddings (128-d) fused into the meta-learner when PyTorch is available
- **Incremental learning** — `partial_fit()` + FIFO replay buffer (200 samples) + EWC regularisation for adaptation without catastrophic forgetting
- **Conformal prediction sets** — 90% coverage-guaranteed uncertainty sets (split-conformal, calibrated on validation fold)
- **Model registry** — JSON+pickle versioning, rollback, A/B metric comparison
- Leakage-free pipeline: imputer/scaler fit only on training indices

### Frontend Inference (predictor.ts)
- 4 expert functions (A/B/C/D) mirror Python expert classifiers
- Attention gating: sharpened softmax (β=2.0) over expert confidences + domain-aware gate adjustment
- Combined uncertainty: 50% normalised entropy + 50% conformal set-size fraction
- Temperature scaling: T = 0.72 (CTU-CHB val calibrated)
- Conformal q̂ = 0.28 (90% coverage)

### Design Preferences
- **Frontend:** React 18 + Vite 5 + TypeScript, port 5000
- **Prediction:** Fully client-side via `src/lib/predictor.ts`
- **Research tab:** Displays real evaluation numbers from `ctu_model_results.json`
- **No external tabular datasets** anywhere in training, evaluation, or display

## Project Architecture

```
├── src/                          # React + Vite frontend
│   ├── App.tsx                   # Main app with tabs and ResearchPanel
│   ├── components/ResultPanel.tsx # Doctor-facing result view
│   └── lib/predictor.ts          # Client-side inference (AdaptiveReserveNet v3)
├── adaptive_reservenet.py        # AdaptiveReserveNet v3.0 — core model class
├── model_registry.py             # Model versioning, rollback, A/B comparison
├── train_adaptive.py             # PRIMARY training entry point (v3)
├── reservenet_model.py           # Backward-compat re-export of AdaptiveReserveNet
├── pulsefm_encoder.py            # CNN-TCN PulseEncoder (optional, needs torch)
├── ctu_loader.py                 # Strict CTU-CHB loader (no synthetic fallback)
├── ctg_feature_engine.py         # Signal feature extraction (FHR/UC → tabular)
├── metrics_utils.py              # Evaluation metrics + bootstrap CIs
├── ctu_model_results.json        # Generated by training — real evaluation numbers
├── models/                       # Trained model artifacts + registry
│   ├── adaptive_reservenet.pkl   # AdaptiveReserveNet v3 bundle
│   └── registry/                 # ModelRegistry versioned store
│       └── registry.json
├── results/                      # Per-run metric snapshots
└── package.json                  # Frontend dependencies
```

## Running the Application

```bash
npm run dev                        # Start React frontend (port 5000)
npm run build                      # Build for static deployment → dist/
python train_adaptive.py           # Train AdaptiveReserveNet v3 + write results JSON
python train_adaptive.py --update models/adaptive_reservenet.pkl  # Incremental update
```

## Key Files

- **adaptive_reservenet.py**: Core v3 model — 4 experts, attention gating, EWC incremental learning, conformal prediction
- **model_registry.py**: Version/rollback management for all trained models
- **train_adaptive.py**: Full training pipeline — CTU-CHB records → AdaptiveReserveNet + TOPQUA ensemble → `ctu_model_results.json`
- **src/lib/predictor.ts**: TypeScript mirror of v3 inference (4 experts, attention weights, conformal uncertainty)
- **pulsefm_encoder.py**: Optional CNN-TCN waveform encoder (torch required)

## Deployment

Deployed as a **static site** — `npm run build` generates `dist/` which is served directly.
Python scripts are research/training only and are not part of the deployment.
