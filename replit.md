# FetalyzeAI - Fetal Health Classification Dashboard

## Overview
A React + Vite dashboard for fetal health classification trained on **real intrapartum waveform data** from the CTU-UHB and CTGDL databases. The model is trained on raw FHR/UC signals — not tabular feature extracts. Run `python train_ctu_signal_model.py` to generate real evaluation metrics into `ctu_model_results.json`.

**Key Feature:** All training uses real hospital-grade waveform data (CTU-UHB, PhysioNet). No Kaggle/UCI tabular datasets.

### Datasets (primary sources only)
- **CTU-UHB Intrapartum CTG Database (PRIMARY)**: 552 real intrapartum CTG recordings (FHR + UC @ 4 Hz). Clinical outcomes: cord blood pH, base deficit, Apgar scores. Source: PhysioNet (Chudáček et al., 2014)
- **CTU-CHB Annotation Dataset (CTGDL)**: Expert morphological annotations — baseline, decelerations, accelerations, uterine contractions. Source: Zenodo 19510407

### Model Architecture
- Regularized XGBoost (depth 3, strong L1/L2, class-balanced)
- 5-member MLP ensemble with temperature scaling for calibrated uncertainty
- Leakage-free pipeline: imputer/scaler fit only on training indices inside each fold
- Primary metrics: held-out test macro-F1, high-risk recall, AUROC

### Design Preferences
- **Frontend:** React 18 + Vite 5 + TypeScript, port 5000
- **Prediction:** Fully client-side via `src/lib/predictor.ts`
- **Research tab:** Displays real evaluation numbers from `ctu_model_results.json`
- **No Kaggle/UCI data** anywhere in training, evaluation, or display

## Project Architecture

```
├── src/                          # React + Vite frontend
│   ├── App.tsx                   # Main app with tabs and ResearchPanel
│   ├── components/ResultPanel.tsx # Doctor-facing result view
│   └── lib/predictor.ts          # Client-side inference
├── train_ctu_signal_model.py     # CTU-UHB primary model training (outputs ctu_model_results.json)
├── ctgdl_loader.py               # CTU-UHB / CTGDL dataset loader
├── ctgdl_features.py             # Signal feature extraction (FHR/UC → tabular features)
├── ctu_model_results.json        # Generated by training script — real evaluation numbers
├── ctu_signal_model.pkl          # Trained model artifact
└── package.json                  # Frontend dependencies
```

## Running the Application

```bash
npm run dev                       # Start React frontend (port 5000)
python train_ctu_signal_model.py  # Generate real evaluation metrics
```

## Key Files

- **src/App.tsx**: Main app with tabs including ResearchPanel showing real CTU-UHB metrics
- **train_ctu_signal_model.py**: Trains on CTU-UHB, outputs `ctu_model_results.json`
- **ctgdl_loader.py**: Loads CTU-UHB via PhysioNet WFDB or Zenodo CSV fallback; synthetic fallback for demo
- **ctgdl_features.py**: Extracts clinical signal features (baseline FHR, STV, LTV, decelerations, etc.)
